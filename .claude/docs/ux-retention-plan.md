# UX 포기 방지 시스템 기획

> 목표: 이직 준비를 포기하지 않도록 "떠먹여주는" 경험 설계
> 작성일: 2026-04-02

## 포기 지점 분석

```
[앱 첫 진입]
    ↓
[캐릭터 생성] ← 포기①: "뭘 만드는 거지? 왜 해야 하지?"
    ↓
[퀘스트 맵] ← 포기②: "뭐부터 해야 하지? 다 어려워 보여"
    ↓
[퀘스트 클릭]
    ↓
[폼 등장] ← 포기③: "뭘 써야 할지 모르겠어"
    ↓
[AI 제출]
    ↓
[결과 수령]
    ├── 통과 → ← 포기④: "다음에 뭐 하지?" (동력 상실)
    └── 실패 → ← 포기⑤: "또 실패네..." (좌절 → 이탈)
         ↓
[다음 날 재접속] ← 포기⑥: "어디까지 했더라?" (컨텍스트 단절)
```

## 개선 기능 목록

### A. 온보딩 스토리텔링 (포기①)
- 캐릭터 생성 전 RPG 인트로 내러티브
- "5년차 백엔드 개발자, 이직을 결심했다..." 스토리
- 현재: 바로 이름/스킬 입력 → 개선: 동기 부여 후 입력

### B. 퀘스트 맵 "오늘의 미션" 배너 (포기②)
- 미완료 퀘스트 중 다음 추천 1개 강조 표시
- 이전 완료 퀘스트와의 연결 고리 설명
- 예: "어제 1-1을 마쳤어요! 이제 이직 동기를 정리해봐요"
- [바로 시작하기 →] CTA 버튼

### C. 퀘스트 브리핑 모달 (포기③)
- 퀘스트 클릭 시 폼 전 브리핑 화면
- 이 퀘스트로 얻는 것 / 예상 소요 시간 / 예시 답변 보기
- [시작하기] 버튼으로 폼 진입

### D. 필드별 작성 가이드 (포기③)
- 각 입력 필드 옆 `?` 버튼 → 작성 팁 슬라이드
- "이렇게 쓰면 높은 점수" 예시 답변 토글

### E. 통과 후 다음 퀘스트 연결 카드 (포기④) — Sprint 1
- AI 결과 통과 시 "다음 퀘스트 바로 시작하기" 버튼
- 이 결과가 다음 퀘스트에 어떻게 활용되는지 설명
- 퀘스트 간 연결 고리 (예: "1-2 에세이가 1-BOSS에 활용됩니다")

### F. 실패 시 재도전 코치 (포기⑤) — Sprint 1
- AI_FAILED 시 구체적 개선 포인트 하이라이트
- "이전 답변 불러와서 수정" 버튼 (처음부터 다시 안 써도 됨)
- 재도전 격려 메시지

### G. 복귀 배너 (포기⑥)
- 마지막 접속 후 N일 경과 시 복귀 배너 노출
- "N일 전 X 퀘스트를 완료했어요. 다음은 Y예요"
- 진행률 요약 + [이어서 하기 →]
- BE: 마지막 접속 시간 기록 필요

## 스프린트 계획

### Sprint 1 — 결과 화면 개선 (FE only, BE 변경 없음)
- **E**: 통과 후 다음 퀘스트 연결 카드
- **F**: 실패 시 재도전 코치 + 이전 답변 불러오기

### Sprint 2 — 진입 장벽 제거 (FE only)
- **B**: 퀘스트 맵 "오늘의 미션" 배너
- **C**: 퀘스트 브리핑 모달

### Sprint 3 — 심화 가이드 + 재접속 (FE + BE 소폭)
- **D**: 필드별 작성 가이드
- **G**: 복귀 배너 (BE: 마지막 접속 시간 기록)

### Sprint 4 — 온보딩 (FE)
- **A**: 온보딩 스토리텔링 인트로

## 퀘스트 간 연결 고리 (E 기능 데이터)

```
1-1 완료 → "1-2에서 진단 결과를 바탕으로 이직 동기를 정리해요"
1-2 완료 → "1-BOSS에서 1-1, 1-2를 종합 분석해요"
1-BOSS 완료 → "ACT II에서 약점 기술을 집중 공략해요"
2-2 완료 → "2-3 시스템 설계에도 같은 기술 깊이가 필요해요"
2-3 완료 → "2-BOSS 모의 면접에서 설계 경험을 어필해요"
2-BOSS 완료 → "ACT III에서 이 기술력으로 맞는 회사를 찾아요"
3-2 완료 → "3-BOSS에서 JD 분석을 바탕으로 최종 타겟을 확정해요"
3-BOSS 완료 → "ACT IV에서 지원 패키지를 완성해요"
4-1 완료 → "4-BOSS에서 이력서와 GitHub를 함께 점검해요"
4-BOSS 완료 → "ACT V에서 실전 지원을 시작해요"
```
