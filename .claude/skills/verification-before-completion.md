# SKILL: Verification Before Completion (완료 전 검증)

**트리거:** 완료 선언, 커밋, PR 생성, "됐다"고 말하기 전 — 반드시

## 철의 법칙

```
신선한 검증 증거 없이는 완료 주장 없다
```

## 검증 게이트

```
완료 주장 전:
1. 무엇을 실행해야 이 주장을 증명하는가?
2. 실행한다
3. 전체 출력을 읽는다
4. 출력이 주장을 뒷받침하는가?
   YES → 증거와 함께 주장
   NO  → 실제 상태를 정직하게 보고
```

## 금지 표현 (검증 전)

- "됐을 것 같아요"
- "아마 통과할 거예요"
- "확인해 보니 잘 되는 것 같습니다"
- "좋아요!", "완료!", "성공!"

## 각 주장별 필요 증거

| 주장 | 필요한 증거 |
|------|------------|
| 테스트 통과 | 테스트 명령 실행 출력: 0 failures |
| 빌드 성공 | 빌드 명령 exit 0 |
| 버그 수정 | 원래 증상 재현 테스트 통과 |
| 요구사항 충족 | 계획 각 항목 체크리스트 확인 |
| API 계약 일치 | BE 스펙 vs FE 타입 직접 대조 |

## 프로젝트 검증 명령어

```bash
# BE 테스트
./gradlew test

# BE 빌드
./gradlew build

# FE 타입 체크
cd fe && npx tsc --noEmit

# FE 빌드
cd fe && npm run build
```
