# SKILL: Merge Quiz (머지 전 이해도 퀴즈)

**트리거:** 사용자가 "퀴즈", "이해했는지 확인" 요청 시, 또는 규모 큰 PR(신규 도메인·마이그레이션·
아키텍처 변경 포함) 머지 전에 오케스트레이터가 제안

## 목적

diff를 눈으로 읽은 것과 동작을 이해한 것은 다르다. 이 스킬은 **"이해했다고 착각하는 상태"**
(unknown unknowns의 마지막 형태)를 머지 전에 드러낸다. 틀린 문제 = 다시 봐야 할 코드.

이 프로젝트는 학습 목적(이직 준비)이므로, 퀴즈는 검증 장치이자 학습 장치다.

## 절차

1. 리뷰 대상 diff/브랜치의 변경을 파악한다 (`git diff origin/main...HEAD --stat` → 핵심 파일 Read)
2. 3~5문제를 만든다. 문제 유형:
   - **동작 예측**: "이 입력이 들어오면 어떤 응답/상태가 되는가?"
   - **경계 조건**: "X가 null/빈 값/중복이면 어디서 어떻게 처리되는가?"
   - **설계 근거**: "왜 A 대신 B 구조를 택했는가?" (Deviations·결정 테이블 기반)
   - **파급 범위**: "이 변경으로 함께 영향받는 기존 기능은?"
3. 각 문제에 **관련 코드 위치(파일:라인)를 링크**한다
4. 한 번에 한 문제씩 낸다. 답을 받은 뒤 정오답 + 근거 코드를 보여준다
5. 종료 시 요약: 맞은/틀린 문제, 틀린 문제의 재검토 파일 목록

## 규칙

- 답을 먼저 보여주지 않는다 — 사용자가 답한 뒤 공개
- 문제는 diff에 실제로 존재하는 동작만 다룬다 (추측성 문제 금지)
- 사소한 문법·네이밍 문제 금지 — 동작·설계·경계 조건만
- 전부 정답이어도 "재검토 파일 없음"을 명시하고 종료

## 종료 조건

전 문제 풀이 완료 + 틀린 문제의 코드 재확인 완료 (또는 사용자가 중단 요청)
