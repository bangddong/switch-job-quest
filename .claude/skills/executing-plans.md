# SKILL: Executing Plans (계획 실행)

**트리거:** `.claude/plans/` 에 작성된 구현 계획 실행 시

## 프로세스

1. **계획 파일 읽기** → 비판적으로 검토 → 질문/우려사항 있으면 먼저 논의
2. **각 태스크 실행:**
   - 태스크 시작 → `in_progress` 표시
   - 단계별 실행 (TDD: RED → GREEN → REFACTOR)
   - 태스크 완료 → 커밋 → `completed` 표시
3. **막히면 즉시 중단하고 도움 요청** (추측하지 않는다)
4. **전체 완료 후:** `verification-before-completion` 스킬 실행

## 태스크 상태 표시

계획 파일에서 체크박스로 추적:
```markdown
- [x] Task 1: 완료
- [ ] Task 2: 진행 중
- [ ] Task 3: 대기
```

## 절대 금지

- main 브랜치에서 직접 구현 시작 (브랜치 생성 후 진행)
- 막혔을 때 추측으로 진행
- 검증 단계 건너뛰기
- 여러 태스크를 한 커밋에 묶기 (태스크마다 커밋)

## 막혔을 때

```
막힌 위치: Task N, Step M
원인: [관찰한 것]
시도한 것: [이미 시도한 접근]
질문: [구체적 질문 하나]
```
